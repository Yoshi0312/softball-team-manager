# 次スプリントチケット（2026-02-20 起票）

## STATS-201: 試合種別フィルタ時にチーム成績サマリが同条件で再集計されない
- 種別: バグ修正
- 優先度: High
- 背景:
  - `stats` 画面で試合種別を変更した際、試合一覧・選手成績・ダッシュボードチャートは種別で絞り込まれる。
  - 一方でチーム成績サマリ表のみ、全種別集計のまま残る可能性がある。
- 現状実装:
  - `renderTeamSummary()` 内の `calculateTeamSummary(state.gameStats, { year: state.selectedYear })` で `type` 未指定。
- 期待仕様:
  - 画面上の全統計表示（サマリ表 / games / players / dashboard）が同じ `year + gameType` 条件に一致すること。
- 受け入れ条件:
  - `changeStatsGameType()` 操作時、チーム成績サマリ表も同じ種別条件で再集計される。
  - `all` 選択時は従来どおり全種別集計になる。
  - 既存の年度絞り込み動作を壊さない。

## QA-118: 手動回帰テスト用の認証済み検証導線を整備する
- 種別: QA基盤整備
- 優先度: Medium
- 背景:
  - 現行の手動テスト手順は「ログイン後フロー」が前提だが、非対話CI/検証環境で利用できるテストアカウントが未共有。
  - そのため、実機のログイン後回帰テストが環境依存でブロックされる。
- 期待仕様:
  - 手動/半自動回帰時に再現可能なログイン導線（検証用アカウント、もしくは Firebase Emulator 連携手順）をドキュメント化する。
- 受け入れ条件:
  - `docs/manual-test-checklist.md` から参照できる形で、検証環境でのログイン手順が1つ以上提供される。
  - 少なくとも `stats` ページのログイン後シナリオを第三者が再現可能。
